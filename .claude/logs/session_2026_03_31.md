# Session Log — 2026-03-31

## Live Volatility & Correlation in Basket Returns

**Type**: Feature
**Files modified**:
- `app/backend/main.py` — `get_basket_returns()`, `mode=period` and `mode=cumulative` paths
- `app/frontend/src/components/BasketSummary.tsx` — value formatting

**What was done**:

### 1. Live overlay for volatility and correlation metrics
- Previously, the basket returns view (bar chart and cumulative line chart) only appended live intraday data for price returns (`Close`). Switching to `metric=volatility` or `metric=correlation` showed no live data for the current day.
- Added live `RV_EMA` computation in both `mode=period` and `mode=cumulative`: uses EMA span=10 formula `alpha * |live_return| + (1-alpha) * prev_rv_ema` from the live basket close and previous day's close/RV_EMA.
- Added live `Correlation_Pct` via `_compute_live_breadth_batch()` (constituent-level 21-day pairwise correlation from ticker data). This function was previously only called in `mode=analogs` and `mode=query`.

### 2. Absolute point difference for vol/corr scales
- Changed vol/corr metrics from percentage change `(last/first - 1)` to absolute difference `(last - first)`. RV_EMA differences are annualized by `sqrt(252) * 100` to show values in annualized percentage points.
- Frontend formatting updated: vol/corr values display as `X.XX pts` instead of `X.XX%`. Replaced `isRawPct` variable with `isPointMetric`.
