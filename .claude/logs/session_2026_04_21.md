## 5. Basket returns and chart reliability fixes

- Fixed `1D` cross-basket returns in `main.py`. Single-day windows were previously requiring an exact row on the selected date, which dropped baskets lacking a same-day row. Added anchor slicing so a one-day window uses the latest available row on or before the selected date plus its prior anchor row.
- Investigated ticker chart failures that were throwing:
  - `Assertion failed: data must be asc ordered by time`
- Root cause: same-day duplicate bars from mixed daily and live rows (for example a cached `YYYY-MM-DD 00:00:00` row plus an intraday live row on the same calendar day).
- Fixed this in two places:
  - backend: normalized/deduped chart dates before returning basket or ticker chart payloads
  - frontend: added a `TVChart.tsx` guard that dedupes incoming rows by calendar day before building any chart series
- This restored ticker chart loading for equities and keeps the chart from crashing even if a stale backend process still emits duplicate same-day rows.
