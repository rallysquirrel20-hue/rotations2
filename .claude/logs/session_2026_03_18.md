## Work Completed

### Goal 9: Multi-Basket Backtesting (15 sub-goals tracked)

#### Completed (11/15):

1. **Searchable target picker** — Replaced `<select>` dropdown with combo box (type-to-filter + scrollable list). Supports baskets (Themes/Sectors/Industries) and individual tickers. Applied to both single-leg BacktestPanel and per-leg MultiBacktestPanel. Filter tabs (All/T/S/I/Tickers), arrow key navigation, auto-scroll.

2. **Size & lev buttons uniform width** — All preset buttons use `flex: 1` for equal sizing. Labels fixed width. Both rows (Size/Lev) span identical total width.

3. **Grid layout** — Single-leg: half-width card in 2-column grid. Multi-leg: 2-column grid for leg cards (up to 6 legs), portfolio settings in top half-width card. Full chart area width. Leg cards grow to content, chart area scrolls.

4. **Buy & Hold strategy** — Added `_build_buy_hold()` backend helper. Single trade spanning full date range. Supported in both single-leg and multi-leg endpoints. Frontend hides Trade Source toggle and skips benchmark calls for Buy_Hold.

5. **Header row heights** — Results header, tab row, and equity toggles row all set to 42px matching accordion headers. Back button uses `summary-tab` class.

6. **Strategy toggle button aesthetics** — Replaced inline styles with `analysis-date-controls` container and `bt-strat-btn` class (equal-width buttons). Timeframe buttons pushed right with flex spacer. Matches cross-basket pane styling.

7. **Constituents overlay fixed (single-leg)** — Added `daily_positions` computation inside equity curve loop. Keyed by integer date index. Includes weight, daily_return, contribution per position.

8. **Constituents overlay fixed (multi-leg)** — Changed from string date keys to integer indices. Added full position detail (weight, daily_return, contribution, leg_target). Post-processing sets combined equity.

9. **Per-leg independent toggle** — Changed from single `showPerLeg` boolean to `Set<number>` for per-leg visibility control.

10. **Per-leg standalone curves** — Per-leg equity curves now show full-portfolio standalone simulation (starts at initial_equity). Combined curve still uses allocated capital. Two parallel simulations per leg.

11. **Equity curves indexed to 0%** — All series rebased to percentage returns from visible window start. Y-axis shows percentages. Breakeven at 0%. Timeframe presets (1Y/3Y/5Y/YTD/All) rebase all curves to 0% from that point.

12. **Removed Equity $ input** — No longer needed since charts show percentage returns.

13. **Path tab columns fixed** — Legend widened to 240px. Added CSS for path-legend-header, path-legend-row, path-legend-col with proper flex layout and spacing.

14. **Path tab added to multi-leg** — Backend returns trade_paths per leg. Frontend merges all leg trades, renders chart with blue-to-pink gradient. Legend has Leg (colored), Ticker (conditional), Date, Chg columns. Sortable.

15. **Leverage as position size multiplier** — Changed from cap-only to multiplier: `entry = equity × pos_size × max_lev`. Applied to both single-leg and multi-leg endpoints.
