### Backtest Accuracy Fixes (Session 2 — 2026-03-18)

#### 1. Quarterly Universe Membership Filtering

**Problem**: Backtesting `basket_tickers` mode used only the latest quarter's universe for the entire backtest period. E.g., backtesting Momentum Leaders breakouts in 2008 would trade MU, KGC, OKLO — tickers from the 2026 Q1 universe, not the 2008 universe.

**Fix**: Both `run_backtest()` and `_build_leg_trades()` in `main.py` now:
- Load the full quarterly universe history via `_get_universe_history()`
- Load the union of all tickers across the backtest date range via `_get_universe_tickers_for_range()`
- Build a sorted `_quarter_bounds` list of (quarter_start, ticker_set) pairs
- Filter each trade entry: skip if the ticker wasn't in the basket during the quarter containing the entry date

Applies to both single-leg and multi-leg backtests.

#### 2. Constituents Overlay Overhaul (All 3 Versions)

Expanded the constituents overlay from 4 columns to 6 sortable columns across all three chart types:

**BacktestPanel (single-leg)** — 6 columns: Ticker, Entry, Ent.Wt, Return, Cur.Wt, Contrib
**MultiBacktestPanel (multi-leg)** — 7 columns: Ticker, Leg, Entry, Ent.Wt, Return, Cur.Wt, Contrib
**TVChart (main chart)** — 6 columns: Ticker, Entry, Ent.Wt, Return, Cur.Wt, Contrib

All columns are sortable (click header to toggle asc/desc).

**Backend changes:**
- `run_backtest()` and multi-leg equity loop: store `entry_weight` (alloc/equity at entry time) in `open_filt` dict, include in `daily_positions` output
- `GET /api/baskets/{name}/candle-detail`: added `first_date` (current stint entry via gap detection), `eod_weight` (drifted end-of-day weight)

**Entry date logic**: For basket constituents (TVChart), the entry date is the start of the ticker's current continuous stint — walks backwards through contribution dates, gap > 5 trading days = new stint. For backtest positions, entry date is the trade open date.

#### 3. Overlay Sizing Fix

Wrapped backtest constituents overlays in a constraint div (`position: absolute; top: 4; bottom: 54; left: 8`) so the overlay cannot exceed the chart area. Lower bound sits just above the x-axis (50px chart padding + 4px margin). Content scrolls when it exceeds the available space.

#### 4. Multi-Leg Buy Hold Default

Changed `showBuyHold` default from `true` to `false` in MultiBacktestPanel, matching single-leg behavior. Clean equity curve on first load.

#### 5. BENCHMARK_TIMING Enabled

Set `BENCHMARK_TIMING = True` in `signals/rotations.py` so the live signals loop prints per-step timing breakdowns (equity_ohlc, breadth, correlation, etc.) for each basket.

### Session 3 — Backtest Overhaul (2026-03-18)

#### 1. Trades Tab Enhancement

**Type**: Change
**What was done**: Added entry_weight, exit_weight, and contribution columns to the trades tab in both single-leg and multi-leg backtests. Backend computes these values during equity curve simulation.
**Result**: Success
**Details**:
- Trades now show position weight at entry, weight at exit, and total contribution to portfolio
- Computed inline during equity curve loop rather than as a post-processing step

#### 2. Leverage Enforcement Fix

**Type**: Bug Fix
**What was done**: Changed exposure tracking from sum(original_allocs) to mark-to-market exposure (equity - cash). Prevents over-allocation when positions appreciate. 100% leverage now correctly caps at 100%.
**Result**: Success
**Details**:
- Previous approach summed original allocation amounts, ignoring position drift
- MTM exposure tracks actual portfolio exposure after appreciation/depreciation
- Fixes edge case where positions could exceed leverage limit after gains

#### 3. Multi-leg Weight Denominator Fix

**Type**: Bug Fix
**What was done**: Fixed daily_positions post-processing to recompute all weights (weight, entry_weight, contribution) relative to combined equity instead of per-leg equity.
**Result**: Success
**Details**:
- Multi-leg backtests were computing weights relative to each leg's standalone equity
- Weights now correctly reflect proportion of the combined portfolio

#### 4. Contribution Calculation Fix

**Type**: Bug Fix
**What was done**: Changed contribution calculation from current_weight * return to entry_weight * return. Avoids double-counting appreciation in the contribution metric.
**Result**: Success
**Details**:
- Using current_weight embedded the position's prior appreciation into the contribution
- entry_weight * return isolates the contribution from the position's original sizing

#### 5. initial_equity Removal

**Type**: Refactor
**What was done**: Removed the $100k initial_equity parameter from both frontend and backend. Hardcoded to 1.0 internally since charts display percentage returns.
**Result**: Success
**Details**:
- Frontend no longer shows an equity input field
- Backend normalizes to 1.0 starting equity
- All return calculations work in percentage terms

#### 6. Unified Backtest Component

**Type**: Refactor
**What was done**: Consolidated BacktestPanel.tsx and MultiBacktestPanel.tsx into a single BacktestPanel component supporting 1+ legs. Removed the Chart tab. Added a Distribution tab with a leg selector. Benchmarks auto-calculate for single-leg only.
**Result**: Success
**Details**:
- Eliminates code duplication between single and multi-leg panels
- Chart tab removed (functionality available elsewhere)
- Distribution tab now has a leg selector for multi-leg analysis
- Benchmark curves only computed when a single target is selected

#### 7. Stats Tab Overhaul

**Type**: Change
**What was done**: Replaced flat stats with a nested format containing Portfolio Stats and Trade Stats sections.
**Result**: Success
**Details**:
- Portfolio Stats: return, CAGR, volatility, max drawdown, Sharpe ratio, Sortino ratio, contribution, inter-leg correlation
- Trade Stats: met criteria, taken, skipped, win rate, avg winner/loser, expected value, profit factor, avg time in winner/loser
- Multi-leg shows inter-leg correlation in portfolio stats

#### 8. Entry Date Format Change

**Type**: Change
**What was done**: Changed constituents overlay entry dates from MM-DD to MM-DD-YYYY format in both TVChart and BacktestPanel. Widened the entry date column from 42px to 80px to accommodate.
**Result**: Success
**Details**:
- Full year now visible in entry date column
- Column width increased from 42px to 80px

#### 9. Alloc Rounding Fix

**Type**: Bug Fix
**What was done**: Removed round(alloc, 2) from daily_positions which caused ~20% errors at the initial_equity=1.0 scale.
**Result**: Success
**Details**:
- At initial_equity=1.0, allocations are small decimals (e.g., 0.05)
- Rounding to 2 decimal places introduced significant errors (0.05 vs 0.0512 = ~2.4% error per position, compounding)
- Removing rounding preserves full precision
