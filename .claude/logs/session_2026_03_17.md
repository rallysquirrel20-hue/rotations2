## 5. Performance: Basket Processing Optimization (Goal #15)
**Type**: Refactor
**Files modified**:
- `signals/rotations.py` — pre-computed returns matrices, correlation vectorization, breadth vectorization, equity OHLC vectorization, contributions merge, extracted `_build_quarter_weights`

**What was done**: Comprehensive performance optimization of the basket processing pipeline. Full rebuild dropped from ~30 min to 7 min 47s (467s), averaging 16.9s per basket across 27 baskets. Six distinct optimizations applied:
1. Pre-computed returns matrices — built close/open/high/low returns as Date×Ticker pivot matrices once before the basket loop, cached as `returns_matrix_500.parquet` with fingerprint
2. Correlation optimization (16-26s → 0.7-1.2s per basket) — replaced O(n²×T) Python for-loop over ~6000 dates with numpy within-window z-score variance decomposition using identity `avg_corr = (n * Var(z_portfolio) - 1) / (n - 1)`
3. Breadth vectorization (4s → 2s each) — replaced Python date loops in `compute_breadth_from_trend` and `compute_breadth_from_breakout` with `_vectorized_quarter_filter` (searchsorted + merge) + single groupby aggregate
4. Equity OHLC vectorization (loop → cumprod) — `compute_equity_ohlc` now uses `(1 + returns).cumprod()` for weight drift, producing contributions as a byproduct at zero additional cost
5. Contributions merged into equity OHLC (15-17s → 0.1s) — eliminated redundant computation; contributions come free from the vectorized equity OHLC path; `_compute_and_save_contributions` kept as fallback for incremental updates
6. Extracted `_build_quarter_weights` — shared weight-building logic pulled out of both `compute_equity_ohlc` and `_compute_and_save_contributions`

**Result**: Success
**Details**:
- All breadth columns: exact match (corr=1.0)
- Correlation_Pct: corr=0.999+, 98.5-100% within ±2
- Contributions: daily weight sums exactly 1.0
- All 3 backend correlation endpoints verified working
