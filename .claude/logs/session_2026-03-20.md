## Issues Fixed

### 1. pm2 crash loop flooding system with Python instances

**Type**: Bug fix
**Files**: `live_loop.py`, `ecosystem.config.js`

`live_loop.py` was crashing when NDU (Norgate Data Updater) wasn't running. pm2 kept restarting it every 5 seconds, creating 100+ overlapping Python processes that flooded the system.

**Fix**:
- Caught `BaseException` in `live_loop.py` to prevent unhandled crashes
- Hardened `ecosystem.config.js` with restart safeguards:
  - 30-second restart delay (`restart_delay: 30000`)
  - Maximum 3 restart attempts (`max_restarts: 3`)
  - 60-second minimum uptime threshold (`min_uptime: 60000`)

### 2. Analogs 1D metrics showing all zeros for Volatility/Correlation/Breadth/Breakout

**Type**: Bug fix
**Files**: `app/backend/main.py`

Live basket parquet files only contain OHLC data, so non-return metrics (volatility, correlation, breadth, breakout) were NaN and got forward-filled, making 1D change always compute to 0.

**Fix**:
- Compute live metrics directly from constituents instead of relying on parquet
- Added `_compute_live_breadth_batch()` — reads signals once for all baskets, computes breadth/breakout/correlation from live ticker prices
- RV_EMA computed from basket close EMA

## Features Added

### 3. Non-return metrics show absolute change (not % change)

**Type**: Enhancement
**Files**: `app/backend/main.py`, `app/frontend/src/components/BasketSummary.tsx`

Metrics like breadth now display absolute change instead of percentage change. For example, breadth going from 60% to 65% shows as +5.0% (not +8.3%). Realized volatility (RV) displayed in annualized format (multiplied by sqrt(252) x 100).
