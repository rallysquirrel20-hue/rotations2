### 2. Fix — Q2 correlation missing from basket charts
**Symptom**: User reported correlation data stopped at 3/31 on basket charts while breadth, breakout, and rvol continued into Q2.

**Root cause** (`signals/build_baskets.py:1081-1085` in `_compute_within_basket_correlation`):
```python
q_data = sub_ret.loc[q_start:]
valid_tickers = [t for t in tickers if q_data[t].notna().sum() >= min_obs]
```
With `min_obs = max(10, int(21*0.70)) = 14` and Q2 2026 having only ~5 trading days elapsed on 4/8, no ticker could have 14 non-NA rows in `q_data` — so the entire quarter was skipped, producing zero correlation rows.

**Fix**: check against `sub_ret` (warmup + current quarter) instead of `q_data`. The per-window `col_valid` gate inside the rolling loop still filters sparse windows correctly. Incremental `_finalize_basket_signals_output` recomputes the full correlation history on each run and will backfill existing parquets on the next `rot-signals` cycle.

### 3. Fix — rot-live crashing on startup
**Symptom**: User reported no live updates showing on the frontend. `pm2 status` showed rot-live with 14+ restarts, status "waiting restart".

**Root cause**: `loop_live.py:11` imports `exchange_calendars`. PM2 uses the system Python 3.14 interpreter at `%LOCALAPPDATA%\Python\pythoncore-3.14-64\python.exe`, which did not have `exchange_calendars` installed. rot-signals and rot-universes had the same problem (7 restarts each).

**Fix**: `pip install exchange_calendars` into the system Python 3.14 (v4.13.2, with toolz, pyluach, korean_lunar_calendar as deps). Stopped and restarted all three loop processes. Verified Cycle 1 ran at 11:43:04 ET and both `live_signals_500.parquet` + `live_basket_signals_500.parquet` were written.

**Open follow-up**: `exchange_calendars` needs to be added to setup docs or a requirements file for the PM2 Python env. The backend venv and PM2 system Python are two separate environments and this packaging gap bit us.

### 4. Fix — industries with `&`/`-`/`/` never got live overlays
**Symptom**: User reported that not all industries showed live 1D returns in cross-basket view, while the live OHLC candles on charts were working.

**Root cause**: `signals/build_baskets.py:_cache_slugify_label` normalizes `"Aerospace & Defense"` → `"Aerospace_and_Defense"` (mapping `&→and`). Three sites in `app/backend/main.py` built slugs from `BasketName` using naive `.replace(' ', '_')` or reverse-mapped `slug.replace('_', ' ')` + `str.endswith`, neither of which handles `&`:
1. `main.py:804` (cross-basket returns `live_closes` loop)
2. `main.py:701-702` (`list_baskets` live overlay)
3. `main.py:1883-1884` (`get_basket_data` single-basket overlay)

Of 17 current-quarter industries in `live_basket_signals_500.parquet`, 8 contain `&` → none matched.

**Fix**: added `_basket_name_to_slug(bname)` helper in `main.py` that mirrors `_cache_slugify_label` exactly (`/→space`, `&→and`, `-→space`, `space→_`). Replaced all three sites. Verified: Aerospace_and_Defense, Integrated_Oil_and_Gas, Movies_and_Entertainment, Transaction_and_Payment_Processing_Services all now have Apr 8 rows in the single-basket endpoint.

### 5. Fix — cross-basket returns bypassed current-quarter universe filter
**Symptom**: User clarified they want cross-basket view to show ONLY current-quarter active industries (17), not all 50 from gics_mappings. When the universe filter in the top-right is active, they want to see industries that met the dollar-volume criteria in any quarter within the selected window.

**Root cause**: `list_baskets`, `/api/baskets/breadth`, `/api/baskets/compositions`, and `/api/signals/log` already use `_is_valid_basket()` which gates on `_get_valid_industry_slugs()` (current quarter only). But `get_basket_returns` at line 765 bypassed this helper and loaded all 50 industry names directly from `industry_u` — inconsistent with the sidebar.

**Fix** (backend `main.py`):
- Added `_industries_for_quarter_range(start_qkey, end_qkey)` helper that reads `industry_u` and unions slugs active in any quarter within the range. Falls back to `_get_valid_industry_slugs()` when both bounds are None.
- Added `universe_start: str`, `universe_end: str` query params to `get_basket_returns`. The `all_slugs` set now uses this helper instead of the raw `industry_u` keys.

**Fix** (frontend):
- Added `universeQuarters?: { start, end } | null` prop on `BasketSummary` and `BasketReturnsChart`.
- `App.tsx` passes `{ start: quarterStart, end: quarterEnd }` when `isQuarterMode`, else `null`.
- `BasketReturnsChart` includes `universe_start`/`universe_end` in cumulative/period axios calls and in the initial date-bounds fetch.
- Added `universeQuarters?.start`/`.end` to both `useEffect` dep arrays for refetch on filter change.

**Verified at backend**: default returns 17 industries (current Q2 2026), `universe_start=2025 Q1&universe_end=2026 Q2` returns 24 (union), explicit `2026 Q2` returns 17.
