## Fix: ticker-signals endpoint 500 error (numpy float32 serialization)

### What changed
After the signals_500.parquet full rebuild, the `/api/ticker-signals` endpoint started returning 500 Internal Server Error. Root cause: `Close` and `Volume` columns are `float32` dtype, and `round()` on numpy float32 returns numpy float32, which FastAPI's JSON encoder cannot serialize.

### Files modified
1. **rotations_app/backend/main.py:676** — Wrapped `pct_change` calculation with `float()` cast
2. **rotations_app/backend/main.py:683-689** — Cast `pct_change` to `float()` and `dollar_vol` to `int()` in result dict
3. **rotations_app/backend/main.py:702** — Wrapped live override `pct_change` with `float()` cast

### Why
Pre-existing latent bug exposed by the full parquet rebuild. numpy float32 values are not JSON-serializable by FastAPI's default encoder.

### Status
Fixed and verified — endpoint now returns 1,257 tickers with all columns populated. However, frontend sidebar still shows empty columns (DV, LT, ST, MR, CHG) despite API returning correct data — needs investigation.
