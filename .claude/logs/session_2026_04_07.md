### Industry Dollar Volume Filter

5. **`config.py`**: Added `INDUSTRY_TOP_PCT = 0.25`, kept `INDUSTRY_MIN_STOCKS = 3` as floor

6. **`build_universes.py` / `_build_gics()`**: After building raw industry mappings, computes total dollar volume per industry per quarter (using `_get_quarterly_volume`). Per quarter: excludes industries with < 3 tickers, then selects top 25% by dollar volume. Different quarters can have different qualifying industries.

7. **`build_baskets.py`**: Daily incremental runs only process current-quarter industries (`current_key in u`). Force rebuilds (`--force`) process all historical industries. Reduced from ~73 to ~38 daily baskets.

8. **`live_updates.py`**: `_get_all_basket_specs_for_reports()` and its industry list now filtered to current-quarter industries only.

9. **`app/backend/main.py`**:
   - Replaced static `MIN_INDUSTRY_TICKERS` / `_get_small_industry_slugs()` with `_get_valid_industry_slugs()` — reads current quarter's industries from pre-filtered GICS file
   - **Compositions endpoint**: removed `_is_valid_basket` filter so full quarterly history is browsable via universe filter
   - **Returns/analogs endpoint**: replaced `_is_valid_basket` with GICS-file check — includes all historically qualifying industries, not just current quarter
   - Sidebar + breadth: still filtered to current quarter only
   - Fixed `&` vs `and` mismatch: added `_slug_to_gics_name()` helper that tries both `"Aerospace and Defense"` and `"Aerospace & Defense"` when looking up GICS names. Applied to `get_latest_universe_tickers()` and `_get_universe_history()`.

### Backtest Basket-Level Filters

10. **`BacktestPanel.tsx`**: Added basket-level metric filters for constituents mode
