# Session Log — 2026-04-01

## Q2 Quarter Transition — Universe Keying Fix

**Type**: Critical bug fix
**Trigger**: First day of Q2 2026 — live signals showed "No universe found for 2026 Q2"

### Root cause
`build_quarter_universe()` in `rotations.py` keyed universes by *source* quarter (Q1 volume data → `"2026 Q1"` key) instead of *target* quarter (Q1 volume data → `"2026 Q2"` key). The live path looked up `"2026 Q2"` directly but it didn't exist. The thematic builders compensated with a `prev_key` offset, creating a confusing double-offset that happened to produce correct results by accident.

### Fix applied to rotations.py
- Offset universe key +1 quarter in `build_quarter_universe()` and `build_quarter_etf_universe()`
- Removed `prev_key` offset from all 8 thematic/sector/industry builders (beta, momentum, risk-adj momentum, dividends, size, volume growth, sectors, industries) — now use `QUARTER_UNIVERSE[key]` directly
- Removed dead `get_universe()` function (was never called)
- Bumped `EQUITY_UNIVERSE_LOGIC_VERSION` to `'2026-04-01-target-quarter-keying'`

### Quarter transition logic added
- After equity curve processing finishes (data through Q1), if Q2 exists in the universe, transition state to Q2 with fresh dollar-volume weights — applied to both vectorized and incremental paths
- Added rebalance contribution rows at Q2 start date so contributions parquet immediately reflects the new basket
- Backend `get_basket_data()` now reads weights from meta (canonical basket state) instead of contributions parquet

### PM2 ecosystem config hardening
- `max_restarts`: 3 → 50 (PM2 was giving up too quickly)
- `min_uptime`: 60s → 300s (live_loop sleeps 15 min between cycles)
- `restart_delay`: 30s → 60s
- Added `kill_timeout: 10s` (prevents overlapping instances when Norgate COM fails)

## Multi-File Architecture — Pipeline Split

**Type**: Major refactoring
**Files created**: `config.py`, `build_universes.py`, `build_signals.py`, `build_baskets.py`, `live_updates.py`

### Motivation
The monolithic `rotations.py` (~9000 lines) entangled universe construction, signal generation, basket processing, and live updates. Quarter transition bugs cascaded because all concerns were coupled. The new architecture separates into 4 independent scripts communicating via disk caches.

### config.py (478 lines)
Shared constants, paths, and utilities imported by all 4 scripts. Zero side effects at import time.
- All constants (SIZE, ETF_SIZE, THEME_SIZE, SIGNALS, version stamps, etc.)
- OutputPaths class, WriteThroughPath, all cache file paths
- Force flags driven by env vars instead of hardcoded booleans
- `DATA_STORAGE_NAME` env var for experiment folder isolation
- `load_universe_from_disk()`, `load_thematic_universe_from_disk(subkey)`, `load_gics_from_disk()`
- `atomic_write_json()`, `atomic_write_parquet()`
- Quarter key helpers, timer utilities, serialization

### build_universes.py (671 lines)
Standalone universe + basket construction. Written fresh, not copy-pasted.
- **NYSE trading calendar** via `exchange_calendars` — precise last-trading-day detection for quarter boundaries
- **Target-quarter keying**: Q1 volume → `"2026 Q2"` key (correct from the start)
- **Delisted stock filter**: removes tickers with Norgate `-YYYYMM` suffix before ranking, so all 500 slots are active stocks
- **Proactive next-quarter build**: when Norgate data covers the last NYSE trading day of the current quarter, the next quarter's universe is built immediately
- **Generic helpers**: `_cached_build()`, `_parallel_fetch()`, `_safe_quarterly()`, `_rank_per_quarter()`, `_collect_metric()` — eliminated 600+ lines of boilerplate from 10 builders
- **All industry baskets** tracked (removed INDUSTRY_MIN_STOCKS=10 filter)
- Verified: momentum leaders, volume growth, beta universes match old output for Q3-Q1; Q2 correctly filters delisted names

### build_signals.py (1021 lines)
Standalone signal generation.
- **Removed 78 PIT rolling stat columns** (Win_Rate, Avg_Winner, Historical_EV, etc. × 6 signals) — only used by intrabasket signal display, now replaced by the backtester
- Parquet reduced from 144 → 66 columns
- `_numba_pass5_signal` stripped to trade tracking only (no accumulator state) — 33% faster full rebuild
- **Unified stock/ETF pipelines**: `_load_or_build()`, `_incremental_update()`, `_cache_is_current()` — eliminated ~400 lines of duplicate code
- Shared helpers: `_optimize_dtypes()`, `_save_signals()`, `_build_keep_set()`, `_progress()`
- Verified: NVDA and LEHMQ-201203 signal columns match old output across all 66 retained columns

### build_baskets.py (1821 lines)
Standalone basket equity/signal generation.
- Removed PIT stats from `_append_trade_rows` (calc_rolling_stats dependency eliminated)
- Removed dead functions: `compute_signal_trades`, `_append_trade_rows`, `_fmt_price`, `_fmt_bars`, `_fmt_pct`
- Removed duplicate `_cache_build_quarter_lookup` / `_cache_find_active_quarter`
- Fixed `load_thematic_universe_from_disk` subkey support for beta (high/low), momentum (winners/losers), dividends (high_yield/growth/with_growth)

### live_updates.py (1472 lines)
Standalone live intraday update script.
- **Removed 580 lines of dead PDF report code** (export_annual_returns, export_last_20_days, all _render_* chart functions, _render_return_table_pages)
- Removed dead `append_live_today_to_signals_parquet`, `export_group_holdings`, `export_current_quarter_universe`
- Removed 7 dead report helper functions
- Stripped PIT stats from live signal exports (Excel output now: Date, Ticker, Close, Signal_Type, Theme, Sector, Industry, Entry_Price)
- Simplified thematic universe loading using subkey-aware `load_thematic_universe_from_disk()`

### Data flow
```
build_universes.py → JSON caches (universes, baskets, GICS)
    ↓
build_signals.py → parquet caches (stock + ETF signals)
    ↓
build_baskets.py → parquet caches (basket equity, signals, contributions)
    ↓
live_updates.py → live parquets (intraday signals, basket OHLC)
    ↓
app/backend/main.py → REST API → frontend
```
