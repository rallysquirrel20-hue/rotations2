- `export_live_today_to_basket_signals_parquet()` was called only within the `if GENERATE_ANNUAL_RETURN_REPORT:` block

## Design Decisions

### Live Data Strategy (Option A)
- `livesignals.py` appends live rows with `Source='live'` directly into consolidated parquets
- Backend reads one file per data type (no merge at query time)
- Idempotent: existing `Source='live'` rows for today dropped before re-append
- Cleaner than Option B (separate live files + merge)

### Split Pipeline Communication
- No direct file I/O between phases (no cross-imports)
- Foundation module contains only signal computation, paths, helpers
- Each phase is independently runnable, restartable, and schedulable

## Performance Impact
- Per-basket rebuild still targets <20s (vectorization optimizations carry forward)
- Full 27-basket rebuild: <10 min cold, <2 min incremental
- Live intraday loop: ~75s per cycle
